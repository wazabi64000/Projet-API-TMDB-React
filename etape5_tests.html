Objectif

Tester les endpoints du serveur pour vérifier que tout fonctionne correctement.

Explications

Utilisez Postman ou le navigateur pour tester les routes GET et vérifier les réponses.

Exemple de code


# Tester l'endpoint principal
GET http://localhost:3000/

# Tester un endpoint de films
GET http://localhost:3000/movies?title=Inception

Bonnes pratiques

  • Tester tous les endpoints avant de connecter le frontend
  • Vérifier les codes HTTP et les messages d'erreur
  • Tester les limites avec rate limiter

Notes et conseils

Vous pouvez ajouter des logs dans les controllers pour mieux comprendre le flux des requêtes.

Résumé

Les tests confirment que le serveur fonctionne, les routes répondent correctement et le backend est prêt pour le frontend React.
