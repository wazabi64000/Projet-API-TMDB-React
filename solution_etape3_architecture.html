1️ Objectif

Mettre en place une architecture MVC (Model-View-Controller) moderne pour notre API Node.js :

  • Controllers : gérer la logique métier et les réponses HTTP
  • Services : interactions avec TMDB et traitement des données
  • Routes : définir les endpoints et connecter controllers
  • Fichier .env pour les clés secrètes

Action : Créez les dossiers suivants dans votre projet : controllers/, services/, routes/.

Métaphore : Imaginez le MVC comme un restaurant : Routes = serveur qui prend la commande, Controllers = chef qui prépare le plat, Services = fournisseurs et ingrédients.

2️ Explications détaillées

Le pattern MVC sépare clairement les responsabilités :

  • Routes = point d’entrée de l’application. Ex: GET /movies
  • Controllers = logiques métiers et décisions. Ex: filtrer ou transformer les données de TMDB
  • Services = appels API externes et traitements spécifiques
  • Env = stocke les clés API de manière sécurisée

Action : Créez un fichier tmdbService.js dans services et un fichier movieController.js dans controllers.

3️ Exemple concret


// services/tmdbService.js
import axios from 'axios';
import dotenv from 'dotenv';
dotenv.config();

export const fetchMovies = async (query) => {
  const url = `https://api.themoviedb.org/3/search/movie?api_key=${process.env.TMDB_KEY}&query=${query}`;
  const response = await axios.get(url);
  return response.data.results;
};

Action : Testez ce service avec Node REPL ou un petit script pour vérifier la récupération des films.

4️ Cas concrets / erreurs fréquentes


// Oublier dotenv
const url = `https://api.themoviedb.org/3/...&api_key=${process.env.TMDB_KEY}`;
npm ERR! process.env.TMDB_KEY is undefined

Solution : créer un fichier .env à la racine et ajouter TMDB_KEY=VOTRE_CLE


// Mauvaise structure
routes.get('/movies', controllerFunction)
Error: controllerFunction is not defined

Solution : importer correctement le controller depuis controllers/movieController.js

5️ Bonnes pratiques

  • Séparer clairement services et controllers
  • Ne jamais mettre la clé TMDB dans le code source
  • Structurer les routes par domaine (movies, users…)
  • Documenter chaque fonction avec JSDoc ou commentaires

Métaphore : Comme dans un restaurant, chaque poste a son rôle, sinon le plat est raté.
