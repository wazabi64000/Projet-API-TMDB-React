1️ Objectif

Tester le fonctionnement complet de l’API :

  • Tester la récupération des films depuis TMDB
  • Tester les routes GET/POST
  • Vérifier les protections Helmet, CORS et Rate Limit

Action : Préparer Postman ou curl pour tester les routes.

Métaphore : C’est comme inspecter chaque pièce d’une maison avant d’y emménager.

2️ Explications détaillées

Tester l’API permet de :

  • Détecter les erreurs avant que l’utilisateur ne les voie
  • Vérifier que les services communiquent bien avec TMDB
  • S’assurer que la sécurité fonctionne correctement

Action : Créez un script test.js ou utilisez Postman pour envoyer des requêtes GET et POST.

3️ Exemple concret


// Test simple avec Node.js
import { fetchMovies } from './services/tmdbService.js';

(async () => {
  const movies = await fetchMovies('Matrix');
  console.log(movies);
})();

Action : Lancez le script avec node test.js et vérifiez que la liste de films apparaît.

4️ Cas concrets / erreurs fréquentes


// Clé TMDB incorrecte
Error: Invalid API key

Solution : vérifier la valeur de TMDB_KEY dans .env


// Problème réseau
Error: getaddrinfo ENOTFOUND api.themoviedb.org

Solution : vérifier la connexion Internet et réessayer.

5️ Bonnes pratiques

  • Tester chaque route indépendamment
  • Vérifier la structure de la réponse JSON
  • Ne jamais exposer de clés API dans les tests publics
  • Documenter les endpoints testés et résultats

Métaphore : Comme un contrôleur qualité : vérifier chaque produit avant de l’envoyer au client.

6️ Résumé et prochaines actions

Vous avez :

  • Vérifié que l’API fonctionne correctement
  • Testé les routes et services
  • Préparé le backend pour être consommé par le frontend React

Action : Passez maintenant au développement du frontend React et à la consommation des données.
