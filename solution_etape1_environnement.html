1️ Comprendre l’objectif

Avant de plonger dans le code, clarifions ce que nous faisons :

  • Créer un dossier propre pour votre projet
  • Initialiser npm pour gérer les modules
  • Vérifier que Node.js fonctionne correctement

Métaphore : C’est comme préparer le chantier : on délimite le terrain, on installe les fondations, et on s’assure que les outils sont fonctionnels.

Action : Ouvrez votre terminal et préparez-vous à créer votre dossier projet.

2️ Créer le dossier et initialiser npm

Tout commence par la création du projet et l’initialisation de npm :


# Créer le dossier projet
mkdir projet-tmdb
cd projet-tmdb

# Initialiser npm
npm init -y

Pourquoi : npm crée le package.json qui servira à installer Express et toutes les dépendances futures.

Action : Tapez ces commandes maintenant et observez que le dossier et le fichier package.json sont créés.

3️ Vérifier Node.js et npm

Avant d’aller plus loin, il est essentiel de s’assurer que Node.js et npm sont opérationnels :


node -v
npm -v

Pourquoi : Si ces versions ne s’affichent pas, votre serveur ne pourra pas exécuter de code Node.js. Métaphore : c’est comme vérifier si le moteur tourne avant de démarrer la voiture.

Action : Notez la version affichée, cela vous aidera si vous rencontrez des problèmes de compatibilité.

4️ Cas concrets et erreurs fréquentes

Voici quelques erreurs que vous pourriez rencontrer :


# npm non installé
$ npm init -y
-bash: npm: command not found

Solution : Installer Node.js depuis nodejs.org ou via nvm.


# dossier existant
$ mkdir projet-tmdb
mkdir: cannot create directory ‘projet-tmdb’: File exists

Solution : Supprimer l’ancien dossier ou choisir un nouveau nom. Action : faites-le immédiatement pour éviter toute confusion.

5️ Bonnes pratiques et enchaînement

  • Créer un dossier dédié, pas sur le bureau
  • Vérifier Node.js à jour et stable
  • Documenter dès maintenant avec README.md
  • Ne pas encombrer le projet avec des modules inutiles

Métaphore : Chaque décision ici est comme poser une pierre dans un mur : chaque pierre bien posée garantit la stabilité de tout le bâtiment.

Action : Créez votre README.md et notez les premières idées du projet.

6️ Résumé et prochaines actions

Vous avez :

  • Un dossier projet propre
  • npm initialisé et fonctionnel
  • Node.js vérifié
  • Une base solide pour installer Express et commencer à coder

Action : Passez à l’étape suivante : installation des modules et création de votre serveur Express.
