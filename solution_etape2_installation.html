1️ Objectif

Installer toutes les dépendances nécessaires pour construire notre API Express sécurisée et fonctionnelle :

  • Express pour le serveur
  • Axios pour appeler TMDB
  • Helmet pour la sécurité
  • CORS pour gérer les requêtes cross-origin
  • express-rate-limit pour limiter les requêtes abusives
  • dotenv pour gérer les variables sensibles

Action : Préparez le terminal, on installe maintenant les modules.

Métaphore : Installer ces modules, c’est comme mettre en place vos outils avant de commencer le chantier : chaque outil a un rôle précis.

2️ Explications détaillées

Chaque module a son importance :

  • Express : structure le serveur et les routes, c’est la charpente du bâtiment.
  • Axios : permet de récupérer des données depuis TMDB, comme une pompe qui amène l’eau vers votre chantier.
  • Helmet : protège votre serveur des attaques courantes, c’est comme un pare-feu physique sur le chantier.
  • CORS : contrôle qui peut communiquer avec votre serveur.
  • express-rate-limit : empêche les abus, comme un vigile qui régule le nombre de visiteurs.
  • dotenv : stocke vos clés secrètes à l’abri des regards.

Action : Comprenez bien le rôle de chaque module avant de l’installer.

3️ Exemple concret


# Installer les modules
npm install express axios helmet cors express-rate-limit dotenv

Explication : Chaque package est téléchargé et enregistré dans package.json. Vérifiez que le dossier node_modules apparaît.

Action : Tapez la commande et vérifiez le résultat.

4️ Cas concrets / erreurs fréquentes


# Node version trop ancienne
npm install express
npm ERR! not compatible with your Node.js version

Solution : mettre à jour Node.js ou utiliser nvm pour gérer les versions.


# Problème de réseau
npm install axios
npm ERR! network error

Solution : vérifier la connexion Internet ou utiliser un miroir npm.

5️ Bonnes pratiques

  • Installer uniquement ce dont vous avez besoin pour l’instant
  • Vérifier la version de chaque package et compatibilité
  • Ne jamais exposer vos clés TMDB dans le code : utiliser .env
  • Documenter chaque installation dans README.md

Métaphore : Installer trop de modules inutiles, c’est comme encombrer un chantier de matériaux superflus : vous risquez de vous perdre.

6️ Résumé et prochaine action

Vous avez :

  • Installé tous les modules nécessaires
  • Compris leur rôle et leur importance
  • Préparé le terrain pour créer le serveur Express et les routes

Action : Passez à l’étape suivante : création de l’architecture MVC et configuration des routes et services.
