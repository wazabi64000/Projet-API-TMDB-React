Objectif

Créer la structure MVC du projet : dossiers services, controllers, routes, et le fichier .env pour la configuration.

Explications

L’architecture MVC permet de séparer les responsabilités :

  • Routes : définissent les endpoints de l’API
  • Controllers : traitent les requêtes et appellent les services
  • Services : contiennent la logique métier et les appels à TMDB
  • .env : stocke les clés API et variables sensibles

Exemple de code


# Création des dossiers
mkdir controllers routes services
touch .env
      

# Exemple fichier routes/movies.js
import express from 'express';
import { getMovies } from '../controllers/moviesController.js';
const router = express.Router();
router.get('/', getMovies);
export default router;

Bonnes pratiques

  • Nommer clairement les dossiers et fichiers
  • Utiliser des import/export ESM
  • Ne pas mettre les clés API en dur, utiliser .env

Notes et conseils

Vérifier que chaque dossier contient un fichier index.js ou l’équivalent si nécessaire pour centraliser les exports.

Résumé

L’architecture MVC est en place, le projet est structuré et prêt pour la configuration du serveur.
